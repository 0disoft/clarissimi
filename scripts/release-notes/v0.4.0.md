# Clarissimi {{VERSION}}

Clarissimi {{VERSION}} adds deterministic shell completion and tightens the boundaries around
provider input, generated Markdown, and concurrent local writes.

## Included

- a new `clarissimi completion <bash|zsh|fish|powershell>` command generated from the maintained CLI
  command descriptor
- static completion output that does not inspect repositories, config, credentials, or the network
- fail-closed limits for changed files, provider evidence items, serialized evidence, and provider
  request bodies
- stronger redaction and structural-field checks before repository evidence reaches a provider
- safer generated Markdown links and contributor rendering for untrusted profile and avatar data
- bounded Windows retry behavior for transient file-open and rename contention
- the regenerated committed GitHub Action runtime bundle

## Unchanged boundaries

- no Action input, output, permission, or workflow-trigger changes
- no automatic maintainer approval
- no dynamic shell completion, installation side effect, repository scan, or credential access
- no npm or workspace-package publication
