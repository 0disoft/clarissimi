# Clarissimi {{VERSION}}

Clarissimi {{VERSION}} adds opt-in status updates on the merged source pull request for proposal
workflows.

## Included

- a new `comment-mode: none|upsert` Action input with a silent `none` default
- one bounded GitHub Actions-owned source pull request comment for `propose`, `stage-draft`, and
  `promote-draft`
- create, update, and unchanged outcomes exposed through `source-comment-action` and
  `source-comment-url`
- marker and actor ownership checks that refuse to overwrite unowned comments
- a bounded 1,000-comment scan that fails closed on incomplete or duplicate managed results
- ambiguous comment-create reconciliation without blindly replaying a POST request
- external-consumer validation covering create, update, unchanged, and no-duplicate behavior

## Unchanged boundaries

- source comments remain opt-in and are disabled for `dry-run` and direct `commit` mode
- no new GitHub permission is required beyond the existing proposal-mode `pull-requests: write`
- comments contain only proposal status and links, not evidence text, assessment prose, provider
  output, patches, scores, rankings, or secrets
- no automatic maintainer approval
- no npm or workspace-package publication
