# Clarissimi {{VERSION}}

Clarissimi {{VERSION}} hardens concurrent proposal reruns and custom provider endpoint transport
without changing Action inputs, permissions, or defaults.

## Included

- concurrent first-run source comments converge to one GitHub Actions-owned managed comment
- an equivalent proposal branch winner is accepted only when it has the same base commit and exact
  generated Git tree
- divergent, differently based, or moving proposal branches continue to fail without another push
- the default provider transport resolves every endpoint address and rejects the endpoint when any
  resolved address is non-public
- the selected public address is pinned for the connection while preserving the original HTTP Host,
  TLS server name, and certificate verification
- provider redirects remain disabled so a validated endpoint cannot redirect into another trust
  boundary
- deterministic concurrency and endpoint-transport regressions plus exact-SHA hosted validation

## Unchanged boundaries

- source comments remain opt-in and owned by the GitHub Actions app marker contract
- proposal mode remains the recommended shared-repository write path
- private-network provider endpoints still require the explicit trusted-endpoint opt-in
- no broader GitHub permission, automatic maintainer approval, or force-push behavior
- no npm or workspace-package publication
